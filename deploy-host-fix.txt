COMMANDS TO FIX THE HOST HEADER ISSUE:

1. Pull the latest changes:
cd ~/Expense-Tracker
git pull origin main

2. Stop and remove the current container:
sudo docker stop expense-tracker-backend
sudo docker rm expense-tracker-backend

3. Rebuild the Docker image with the fix:
cd ~/Expense-Tracker/server
sudo docker build -t expense-tracker-backend .

4. Run the container with the updated configuration:
sudo docker run -d \
  --name expense-tracker-backend \
  --restart unless-stopped \
  -p 8080:8080 \
  -e SPRING_PROFILES_ACTIVE=prod \
  -e JWT_SECRET=myProductionSecretKey123456789012345678901234567890 \
  -e DATABASE_URL=jdbc:h2:file:/app/data/expense_tracker \
  -e DATABASE_USERNAME=sa \
  -e DATABASE_PASSWORD=password \
  -e ALLOWED_ORIGINS=https://expense-tracker.kundanprojects.space,http://expense-tracker-api.kundanprojects.space \
  expense-tracker-backend

5. Test the domain access:
curl http://expense-tracker-api.kundanprojects.space/api/health

This should fix the "Host is not valid" issue and allow your custom domain to work properly!
